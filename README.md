## What Is This?

Every company has a unique financial fingerprint — a set of common-size ratios derived from its income statement and balance sheet that reveal its cost structure, asset composition, and profitability profile. Financial DNA is a machine learning application that reads those ratios and answers two questions: what industry does this company belong to, and who are its closest financial peers?

The app takes any stock ticker, pulls its most recent annual financials, computes 10 normalized ratios, and runs them through a suite of trained ML classifiers. The result is a predicted industry sector, a confidence breakdown across all sectors, a radar chart comparing the company's profile to its industry average, and a ranked list of the most financially similar companies in the database.

The core insight is that financial structure is industry-specific. A technology company looks fundamentally different from a utility on a balance sheet — high gross margins, low inventory, minimal PP&E versus heavy fixed assets, high debt, and thin margins. These patterns are consistent enough that models trained on 10 ratios alone can correctly identify a company's sector roughly 80% of the time.

## The 10 Financial Ratios

| Ratio | Formula | What It Signals |
|---|---|---|
| COGS / Revenue | Cost of Goods Sold ÷ Revenue | Production cost intensity |
| Gross Margin | Gross Profit ÷ Revenue | Core product profitability |
| SG&A / Revenue | SG&A Expense ÷ Revenue | Overhead and sales spend |
| Net Margin | Net Income ÷ Revenue | Bottom-line profitability |
| Cash / Assets | Cash ÷ Total Assets | Liquidity cushion |
| Receivables / Assets | Receivables ÷ Total Assets | Credit-based business model |
| Inventory / Assets | Inventory ÷ Total Assets | Inventory intensity |
| PP&E / Assets | Net PP&E ÷ Total Assets | Capital intensity |
| Debt / Assets | Total Debt ÷ Total Assets | Financial leverage |
| Equity / Assets | Equity ÷ Total Assets | Equity financing reliance |

## Repository Structure — `financial_dna_app2/`

### `scripts/`
Contains the entry point and data/training pipeline scripts. `About.py` is the Streamlit home page — launch the app by running `streamlit run scripts/About.py` from the `financial_dna_app2/` directory. `build_dataset.py` constructs the training dataset either by pulling live financials for S&P 500 companies via yfinance (when internet is available) or by generating a synthetic dataset based on known sector financial profiles as a fallback. `train_models.py` reads the dataset and trains all ML models, saving each as a `.pkl` file in the `models/` folder.

### `pages/`
The four pages of the Streamlit app, each handling a distinct feature. `1_Industry_Classifier.py` takes a ticker input and returns a predicted sector, a probability breakdown across all industries, a radar chart comparing the company's ratios to the sector average, and a feature importance chart showing which ratios drove the prediction. `2_Peer_Finder.py` finds the N most financially similar companies using KNN distance in ratio space, displaying a ranked peer table and a PCA scatter plot of the full company universe with the target and its peers highlighted. `3_Company_Comparison.py` accepts two tickers and produces a side-by-side head-to-head comparison with a dual radar chart, a similarity score, and a ratio-by-ratio delta table. `4_Model_Performance.py` displays accuracy and F1 scores for all models, interactive confusion matrices, per-sector classification reports, and feature importance charts across all three tree-based models.

### `models/`
Trained model artifacts saved as `.pkl` files. These are the direct output of running `train_models.py` and are loaded by the app at startup. The folder contains the Random Forest (`rf_model.pkl`), XGBoost (`xgb_model.pkl`), Logistic Regression (`logreg_model.pkl`), KNN (`knn_model.pkl`), SVC (`svc_model.pkl`), and Ensemble (`ensemble_model.pkl`) classifiers, along with supporting artifacts: `scaler.pkl` (StandardScaler fitted on training data), `label_encoder.pkl` (maps numeric predictions back to sector names), `pca_model.pkl` (used for the 2D company map in the Peer Finder), and `metrics.pkl` (accuracy, F1, confusion matrices, and feature importances for all models).

### `data/`
Contains `company_ratios.csv`, the training dataset used by all models. Each row represents one company with its ticker, name, sector, sub-industry, and the 10 ratio values. This file is generated by `scripts/build_dataset.py` — either from live yfinance data or the synthetic fallback — and is the sole source of training data for all ML models.

### `utils/`
Shared utility modules used across all pages. `ratio_engine.py` defines the 10 ratio names and labels and handles computing ratios from raw yfinance financial statement data. `model_manager.py` loads all `.pkl` artifacts and exposes clean functions for prediction and peer finding that the pages import. `data_fetcher.py` handles all yfinance interactions — fetching live company financials and the S&P 500 ticker list. `charts.py` contains all Plotly chart builders: radar charts, dual radar charts, the PCA scatter plot, confusion matrix heatmaps, and feature importance bar charts.

## How to Run

```bash
cd "/path/to/Financial-DNA-tool-/financial_dna_app2"
pip install -r requirements.txt
streamlit run scripts/About.py
```

To rebuild the training data and retrain all models:

```bash
python scripts/build_dataset.py
python scripts/train_models.py
```
